# Оптимизация таймингов Auto-Zapret

## Дата оптимизации: 2026-03-23

## Методология расчёта

Формула: **max(измеренное) × 1.33 = оптимальное** (запас 33%)

Где:
- **max(измеренное)** - максимальное значение из всех успешных тестов
- **1.33** - коэффициент запаса (33% дополнительно к максимуму)

## Измеренные значения (15 тестов, 3 домена)

| Параметр | Максимум | Формула | Оптимальное |
|----------|----------|---------|-------------|
| **Start** (инициализация) | 367ms | (367-50) × 1.33 / 1000 | 0.42s |
| **Test** (тест) | 283ms | 283 × 1.33 / 1000 | 0.38s |
| **Kill** (остановка) | 102ms | 102 × 1.33 / 1000 | 0.135s |

### Детализация теста

| Компонент | Доля | Формула | Значение |
|-----------|------|---------|----------|
| CONNECT | 33% | 283 × 0.33 × 1.33 / 1000 | 0.12s |
| TLS | 50% | 283 × 0.50 × 1.33 / 1000 | 0.19s |
| HTTP | 17% | 283 × 0.17 × 1.33 / 1000 | 0.06s |

## Переключение режимов

В `autozapret/strategy_tester.py` добавлен флаг для быстрого переключения:

```python
AGGRESSIVE_TIMEOUT = True  # Агрессивные тайминги (×5-10 ускорение)
# AGGRESSIVE_TIMEOUT = False  # Консервативные тайминги (для стабильности)
```

**См. также:** `AGGRESSIVE_MODE.md` - подробная документация по переключению режимов.

## Будущая перекалибровка

При изменении условий (новый WinDivert, другая сеть, обновлённый winws) рекомендуется:

1. Запустить тестирование с текущими таймингами
2. Собрать новую статистику (минимум 15 тестов)
3. Пересчитать по формуле: `max × 1.33`
4. Обновить константы в коде

## Скрипт для перекалибровки

```bash
python test_force/ultra_fast.py
```

Скрипт автоматически:
1. Тестирует стратегии для 3 доменов
2. Выводит таблицу всех измерений
3. Расчитывает оптимальные тайминги
4. Показывает готовый код для вставки
