# Изменения в UI: Неблокирующий брутфорс

## Что изменилось

### 1. Модалка брутфорса теперь закрываемая

Раньше:
- Модальное окно было блокирующим (`data-bs-backdrop="static"`)
- Нельзя было закрыть во время выполнения
- Кнопка закрытия останавливала брутфорс

Теперь:
- ✅ Можно закрыть модалку кнопкой [Закрыть] или крестиком
- ✅ Брутфорс продолжается в фоне
- ✅ Прогресс отображается на странице **Логи** (`/logs`)
- ✅ Есть ссылка "Открыть логи с прогрессом" прямо в модалке

### 2. Улучшенное отображение прогресса на странице логов

Страница `/logs` теперь показывает детальную информацию:

```
┌─────────────────────────────────────────────────────────────┐
│ ✅ success  gateway.discord.gg    [first_working]     45/657│
│ ━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━ 6%                           │
│ Текущая: --dpi-desync=multidisorder --dpi-desync-split...  │
│ Фаза: 3                                                       │
│ Всего тестов: 45 | Найдено: 1 рабочих                       │
│ Начато: 07:29:50 | Завершено: 07:30:15                      │
└─────────────────────────────────────────────────────────────┘
```

**Отображаемая информация:**
- 📊 Статус (in_progress / success / failed / stopped_by_user)
- 🏷️ Режим (first_working / all_best)
- 📈 Прогресс бар с процентами
- 🔤 Текущая стратегия (первые 80 символов)
- 📍 Фаза тестирования
- 📊 Всего тестов / Найдено рабочих
- ⏰ Время начала и завершения

### 3. Автоматическое обновление

- Страница логов обновляется **каждую секунду**
- Прогресс синхронизируется между модалкой и страницей логов
- При успехе - автозакрытие модалки через 3 секунды с reload страницы

## Как использовать

### Сценарий 1: Запуск брутфорса с закрытием модалки

1. Откройте `/domains`
2. Нажмите **"Брутфорс"** для нужного домена
3. Откроется модалка с прогрессом
4. Нажмите **"Закрыть"** или крестик
5. Перейдите на страницу **[Логи](http://localhost:8000/logs)** (или откройте в новой вкладке)
6. Наблюдайте за прогрессом в секции **"Активный подбор стратегий"**

### Сценарий 2: Мониторинг нескольких брутфорсов

Если запущено несколько переборов одновременно:

1. Откройте `/logs`
2. В секции "Активный подбор стратегий" будут отображаться **все** активные процессы
3. Каждый показывает:
   - Домен
   - Прогресс
   - Статус
   - Текущую стратегию
4. Можно остановить любой кнопкой 🛑

### Сценарий 3: Длительный брутфорс в фоне

1. Запустите брутфорс
2. Закройте модалку
3. Продолжайте работать с другими разделами
4. Периодически проверяйте `/logs`
5. При успехе - страница автоматически обновится

## Технические детали

### Модалка

**HTML:**
```html
<div class="modal fade" id="bruteForceProgressModal" tabindex="-1">
  <!-- Убрано data-bs-backdrop="static" -->
  <button class="btn-close" onclick="closeBruteForceModal()"></button>
  <a href="/logs" target="_blank">Открыть логи с прогрессом</a>
</div>
```

**JavaScript:**
```javascript
function closeBruteForceModal() {
    // Просто закрываем модалку
    if (bruteForceModalInstance) {
        bruteForceModalInstance.hide();
    }
    // Опрос прогресса продолжается!
}
```

### Страница логов

**Функция `loadProgress()`:**
- Запрашивает `/api/progress` каждую секунду
- Фильтрует активные и завершённые процессы
- Отображает карточки с детальной информацией

**Отображаемые статусы:**
- `in_progress` ⏳ - выполняется
- `success` ✅ - найдена стратегия
- `failed` ❌ - не найдено стратегии
- `stopped_by_user` 🛑 - остановлено пользователем

## API

### GET /api/progress

Возвращает прогресс всех активных брутфорсов:

```json
{
  "progress": {
    "gateway.discord.gg": {
      "domain": "gateway.discord.gg",
      "total": 657,
      "current": 45,
      "status": "in_progress",
      "mode": "first_working",
      "current_strategy": "--dpi-desync=multidisorder...",
      "current_phase": 3,
      "total_tested": 45,
      "total_found": 1,
      "started": "2026-03-25T07:29:50",
      "completed": null
    }
  }
}
```

### POST /api/brute_force/stop/{domain}

Останавливает брутфорс для домена.

## Интеграция с существующим функционалом

### Автоматический брутфорс

Когда Analyzer автоматически запускает брутфорс (при достижении threshold):

1. Прогресс отображается на `/logs`
2. Не требует открытия модалки
3. Пользователь видит в реальном времени

### Принудительный брутфорс через UI

Когда пользователь запускает через кнопку "Брутфорс":

1. Открывается модалка (можно закрыть)
2. Прогресс дублируется на `/logs`
3. После успеха - автозакрытие и reload
