# Режимы брутфорса стратегий

## Обзор

Auto-Zapret теперь поддерживает два режима подбора стратегий для доменов:

### 1. `first_working` (быстрый режим)

**Принцип работы:**
- Тестирование стратегий прекращается при обнаружении первой рабочей
- Стратегии тестируются в порядке приоритета blockcheck.sh
- Минимальное время подбора

**Преимущества:**
- ⚡ Быстрое нахождение рабочей стратегии
- 🕒 Экономия времени (особенно для разблокированных доменов)
- 🎯 Рекомендуется для большинства случаев

**Недостатки:**
- 📉 Может не найти оптимальную по скорости стратегию

**Когда использовать:**
- Приоритет - скорость подбора
- Для доменов с простой блокировкой
- В продакшене по умолчанию

### 2. `all_best` (полный режим)

**Принцип работы:**
- Тестируются ВСЕ стратегии (~634 TLS + ~23 QUIC)
- Собираются метрики скорости (RTT) для каждой рабочей стратегии
- Выбирается стратегия с минимальным временем ответа

**Преимущества:**
- 🏆 Находит оптимальную по производительности стратегию
- 📊 Полный анализ всех вариантов
- ⚙️ Лучшая скорость соединения в будущем

**Недостатки:**
- 🐌 Значительно дольше (может занять 10-30 минут)
- 💻 Больше нагрузка на CPU

**Когда использовать:**
- Приоритет - максимальная производительность
- Для часто посещаемых доменов
- При тонкой настройке системы

## Настройка

### Через конфигурационный файл

Файл: `config/autozapret.json`

```json
{
    "brute_force_mode": "first_working",
    "brute_force_quick_mode": true
}
```

**Параметры:**
- `brute_force_mode`: `"first_working"` или `"all_best"`
- `brute_force_quick_mode`: `true`/`false` для быстрого режима

### Через Web UI

1. Откройте страницу **Домены** (`/domains`)
2. Нажмите кнопку **"Режим брутфорса"**
3. Выберите нужный режим
4. Нажмите **"Сохранить"**

## API Endpoints

### Получить текущий режим

```bash
GET /api/config/brute-force-mode
```

**Ответ:**
```json
{
    "mode": "first_working",
    "quick_mode": true
}
```

### Установить режим

```bash
POST /api/config/brute-force-mode?mode=all_best
```

**Ответ:**
```json
{
    "success": true,
    "mode": "all_best",
    "quick_mode": false
}
```

### Принудительный брутфорс

```bash
POST /api/actions/force-bruteforce/{domain}?mode=first_working
```

**Ответ при успехе:**
```json
{
    "success": true,
    "message": "✅ Найдена стратегия: Phase 3 strategy: fake_multisplit",
    "params": "--dpi-desync=fake,multisplit --dpi-desync-split-pos=method+2"
}
```

### Перевыбор из существующих

```bash
POST /api/actions/reselect-existing/{domain}
```

**Ответ при успехе:**
```json
{
    "success": true,
    "message": "✅ Стратегия перевыбрана: strategy_abc123",
    "strategy": "strategy_abc123",
    "response_time": 0.245
}
```

## Web UI функции

### Страница доменов

На странице `/domains` доступны новые кнопки для каждого домена:

#### 🔀 Перевыбрать
Тестирует все **существующие** стратегии на домене и выбирает лучшую по скорости (RTT).

**Что происходит:**
1. Получаем все стратегии из БД
2. Тестируем каждую на домене
3. Сортируем работающие по RTT
4. Применяем стратегию с минимальным RTT
5. Перезапускаем winws

**Время выполнения:** ~1-5 минут (зависит от количества стратегий)

#### ⚡ Брутфорс
Запускает **полный перебор** стратегий blockcheck-style.

**Что происходит:**
1. Проверка IP блокировки
2. Проверка QUIC поддержки
3. Тестирование стратегий по фазам (1-11)
4. При успехе - создание/применение стратегии
5. При неудаче - cooldown 30 минут

**Время выполнения:**
- `first_working`: 30 секунд - 5 минут
- `all_best`: 10-30 минут

#### 🗑️ Удалить
Удаляет домен из системы.

## Индикация прогресса

При запуске брутфорса открывается модальное окно с прогрессом:

- **Прогресс бар:** показывает % выполнения
- **Текущая стратегия:** отображает тестируемые параметры
- **Фаза:** номер текущей фазы (1-11)
- **Статус:** in_progress / success / failed / stopped_by_user

Можно остановить кнопкой **"Остановить"**

## Примеры использования

### Сценарий 1: Быстрая разблокировка

```
1. Режим: first_working
2. Домен добавлен → автоматический брутфорс
3. Через 1-2 минуты стратегия найдена
4. Сайт работает
```

### Сценарий 2: Оптимизация производительности

```
1. Режим: all_best
2. Домен добавлен → полный перебор
3. Через 15 минут найдено 23 рабочих стратегии
4. Выбрана с RTT 180ms (вместо 450ms у первой найденной)
5. Сайт работает быстрее
```

### Сценарий 3: Перевыбор при деградации

```
1. Старая стратегия работает медленно
2. Кнопка "Перевыбрать" на домене
3. Протестированы существующие стратегии
4. Найдена лучше на 40% быстрее
5. Применена новая стратегия
```

## Технические детали

### Количество стратегий

**TLS (HTTPS):** ~634 стратегий
- 9 позиций сплита × различные комбинации режимов
- Фазы 1-11: basic, fake, disorder, split, fooling, etc.

**QUIC (HTTP/3):** 23 стратегии
- fake repeats
- TTL
- fooling
- ipfrag2

**Итого:** ~657 тестов на домен в режиме `all_best`

### Фазы тестирования

1. **Phase 1-2:** Basic splits
2. **Phase 3:** Multisplit/Multidisorder + Fake с TTL
3. **Phase 4:** Fake с fooling (md5sig, badseq, etc.)
4. **Phase 5:** Fake с AutoTTL
5. **Phase 6:** Orig параметры
6. **Phase 7:** Dup параметры
7. **Phase 8:** WSSIZE полный
8. **Phase 9:** SeqOvl
9. **Phase 10:** Syndata
10. **Phase 11:** IPv6 специфичные

### Метрики скорости

Для каждой стратегии измеряется:
- **RTT (Response Time):** время ответа в миллисекундах
- **Success Rate:** процент успешных подключений
- **Total Checks:** количество проверок

## Troubleshooting

### Брутфорс не находит стратегию

**Возможные причины:**
1. **IP блокировка** - DPI стратегии бесполезны
2. **DNS блокировка** - используйте DoH/DoT
3. **QUIC блокировка** - только TLS стратегии
4. **Сложный DPI** - нужны кастомные fake файлы

**Решения:**
- Проверьте доступность IP через `curl -v https://domain.com`
- Попробуйте другие DNS серверы
- Обновите fake файлы в `files/fake/`

### Слишком долго работает

**Причина:** Режим `all_best` с большим количеством стратегий

**Решения:**
- Переключитесь на `first_working`
- Остановите текущий брутфорс кнопкой "Остановить"
- Используйте "Перевыбрать" вместо полного брутфорса

### Ошибки при рестарте winws

**Симптомы:** Стратегия найдена, но не применяется

**Решения:**
- Проверьте логи winws
- Убедитесь что нет других процессов winws
- Перезапустите Auto-Zapret
