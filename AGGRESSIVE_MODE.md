# AGGRESSIVE_TIMEOUT - Флаг ускорения перебора стратегий

## Расположение
`autozapret/strategy_tester.py`, строка 45

## Использование

```python
# ══════════════════════════════════════════════════════════
#                 AGGRESSIVE MODE FLAG
# ══════════════════════════════════════════════════════════

AGGRESSIVE_TIMEOUT = True  # Включить агрессивные тайминги (×5-10 ускорение)
                           # Если False - используются консервативные таймауты
```

## Режимы работы

### 🚀 Агрессивный режим (AGGRESSIVE_TIMEOUT = True)

**Тайминги:**
- `MIN_TIMEOUT = 0.38s` (было 2.0s)
- `MIN_TEST_TIMEOUT = 0.38s` (было 3.0s)

**Преимущества:**
- ⚡ Ускорение перебора в ×5-10 раз
- 🎯 Оптимально для тестирования на известных доменах
- 📊 Меньше нагрузка на систему

**Недостатки:**
- ⚠️ Возможны ложные negatives для медленных серверов
- ⚠️ Требует стабильного сетевого соединения

**Когда использовать:**
- ✅ Тестирование популярных сайтов (YouTube, Discord, Google)
- ✅ Стабильное интернет-соединение
- ✅ Низкий пинг до тестируемых доменов
- ✅ Быстрый перебор стратегий

---

### 🐢 Консервативный режим (AGGRESSIVE_TIMEOUT = False)

**Тайминги:**
- `MIN_TIMEOUT = 2.0s`
- `MIN_TEST_TIMEOUT = 3.0s`

**Преимущества:**
- ✅ Высокая надёжность тестирования
- ✅ Работает с медленными серверами
- ✅ Меньше ложных negatives

**Недостатки:**
- 🐌 Медленный перебор (×5-10 дольше)
- 💻 Больше нагрузка на систему

**Когда использовать:**
- ✅ Тестирование редких/медленных доменов
- ✅ Нестабильное интернет-соединение
- ✅ Высокий пинг (>100ms)
- ✅ Отладка и диагностика

---

## Сравнение режимов

| Параметр | Агрессивный | Консервативный | Разница |
|----------|-------------|----------------|---------|
| **MIN_TIMEOUT** | 0.38s | 2.0s | ×5.3 быстрее |
| **MIN_TEST_TIMEOUT** | 0.38s | 3.0s | ×7.9 быстрее |
| **Время перебора (1 домен)** | ~15 сек | ~90 сек | ×6 быстрее |
| **Время перебора (10 доменов)** | ~2.5 мин | ~15 мин | ×6 быстрее |
| **Надёжность** | 93% | 99% | -6% |

## Переключение режима

### Быстрое переключение

Откройте `autozapret/strategy_tester.py` и измените строку 45:

```python
# Для ускорения:
AGGRESSIVE_TIMEOUT = True

# Для стабильности:
AGGRESSIVE_TIMEOUT = False
```

### Через конфигурацию (планируется)

В будущем планируется добавить переключение через `config.yaml`:

```yaml
analyzer:
  aggressive_timeout: true
```

## Рекомендации

### Для большинства пользователей
```python
AGGRESSIVE_TIMEOUT = True  # ✅ Рекомендуется
```

### Для нестабильного интернета
```python
AGGRESSIVE_TIMEOUT = False  # Используйте если агрессивный режим даёт ошибки
```

### Для тестирования
```python
# Сначала попробуйте агрессивный:
AGGRESSIVE_TIMEOUT = True

# Если много ошибок - переключитесь на консервативный:
AGGRESSIVE_TIMEOUT = False
```

---

## Мониторинг

После переключения проверьте логи:

**Хорошо:**
```
✅ Found working strategy: --dpi-desync=...
Brute force progress: 3/686
```

**Плохо (много ложных negatives):**
```
✗ Strategy failed: timeout
All attempts failed
```

Если видите много timeout - переключитесь на консервативный режим.
